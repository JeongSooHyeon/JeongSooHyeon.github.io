# 웹 서버와 WAS, Spring Bean 등록 방법 - TIL

## 웹 서버 (Web Server)

### 정의

Static 컨텐츠를 제공하는 서버입니다.

**Static 컨텐츠**
- Image, HTML, CSS 등
- 만들어놓은 그대로 제공
- 어떤 사용자라도 같은 요청을 했을 때 모두에게 항상 동일한 정보를 제공

### 동작 방식

1. 클라이언트가 요청한 정적인 컨텐츠를 HTTP를 통해 제공
2. 동적인 요청을 처리할 때는 컨테이너에게 도움을 요청
3. 컨테이너에게 동적인 요청을 넘겨주면 컨테이너에서 처리한 결과를 받음
4. 받은 결과를 클라이언트에게 제공

---

## 관련 개념

### 컨테이너 (Container)

서블릿의 생명 주기를 관리하고 JSP를 서블릿으로 변환하는 기능을 지닌 프로그램입니다.

### 서블릿 (Servlet)

자바를 사용하여 웹을 만들기 위해 필요한 기술로, 클라이언트의 요청에 대한 응답을 전송하는 역할을 수행합니다.

### JSP (Java Server Pages)

HTML 내에 자바 코드를 삽입하여 웹 서버에서 동적으로 웹 페이지를 생성하여 웹 브라우저에 돌려주는 서버 사이드 스크립트 언어입니다.

---

## WAS (Web Application Server)

### 정의

Dynamic 컨텐츠를 제공하는 서버입니다.

**Dynamic 컨텐츠**
- 요청에 따라 추가적인 데이터 처리가 이루어진 컨텐츠 제공
- 사용자와 그 요청에 따라서 서버 로직 내부에서 데이터 가공이 처리된 후 반환

### 구성

**WAS = Web Server + Web Container**

### 주요 기능

1. 웹 서버로부터 동적인 요청을 받아 처리
2. 응답 결과를 웹 서버에게 다시 돌려줌
3. 웹 서버는 받은 결과를 클라이언트에게 보여줌
4. 프로그램 실행 환경과 데이터베이스 접속 기능 제공
5. 여러 개의 트랜잭션 관리
6. 업무를 처리하는 비즈니스 로직 수행

### 별칭

웹 컨테이너 혹은 서블릿 컨테이너라고도 불립니다.

### 데이터베이스 연동

주로 DB 서버와 함께 수행됩니다.

## Tomcat의 역할

**Tomcat은 WAS에 해당합니다.**

Spring Boot의 내장 톰캣도 WAS로서 동작하며, 정적 컨텐츠 제공과 동적 컨텐츠 처리를 모두 수행합니다.

## Spring Bean 등록 방법

### 1. 컴포넌트 스캔과 자동 의존관계 설정

#### 방법

클래스 선언부 위에 `@Component` 애노테이션 또는 그 하위 애노테이션을 사용합니다.

```java
@Controller
public class UserController { }

@Service
public class UserService { }

@Repository
public class UserRepository { }
```

**@Component의 하위 애노테이션**
- `@Controller`
- `@Service`
- `@Repository`

#### 동작 원리

해당 어노테이션으로 등록된 클래스들은 스프링 컨테이너에 의해 자동으로 생성되어 스프링 빈으로 등록됩니다.

#### 장점
- 간단합니다.

#### 단점
- 빈을 변경해야 할 일이 생기면 일일이 변경해줘야 합니다.

---

### 2. 자바 코드로 직접 스프링 빈 등록

#### 방법

자바 설정 클래스를 만들어 사용합니다.

```java
@Configuration
public class AppConfig {
    
    @Bean
    public UserService userService() {
        return new UserService(userRepository());
    }
    
    @Bean
    public UserRepository userRepository() {
        return new UserRepository();
    }
}
```

**절차**
1. 설정 클래스 선언부 위에 `@Configuration` 애노테이션 추가
2. 특정 타입을 리턴하는 메서드에 `@Bean` 애노테이션 추가

#### 장점
- 빈을 변경할 때 `@Configuration`에 등록된 Bean만 수정하면 됩니다.
- 수정에 용이합니다.

#### 단점
- 설정 코드를 별도로 작성해야 합니다.

## 비교 정리

### 웹 서버 vs WAS

| 구분 | 웹 서버 | WAS |
|------|---------|-----|
| 컨텐츠 | Static | Dynamic |
| 처리 | 정적 파일 제공 | 비즈니스 로직 수행 |
| 구성 | 웹 서버 | 웹 서버 + 웹 컨테이너 |
| DB 연동 | X | O |
| 예시 | Apache, Nginx | Tomcat, JBoss |

### Bean 등록 방법 비교

| 방법 | 장점 | 단점 | 사용 시점 |
|------|------|------|-----------|
| Component 스캔 | 간단함 | 변경 시 일일이 수정 | 대부분의 경우 |
| Java Config | 수정 용이 | 설정 코드 필요 | 환경별 빈 교체 필요 시 |

---

## 핵심 정리

- **웹 서버**: 정적 컨텐츠 제공
- **WAS**: 동적 컨텐츠 제공, 웹 서버 + 웹 컨테이너
- **Tomcat**: WAS에 해당
- **Bean 등록**: Component 스캔(간단), Java Config(유연)
