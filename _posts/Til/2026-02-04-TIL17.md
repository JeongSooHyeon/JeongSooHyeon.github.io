# HTTP 프로토콜 - TIL

## HTTP 프로토콜의 특징

### 비연결성 (Connectionless)
한 번 데이터를 주고 받으면 연결을 끊습니다.

### 무상태성 (Stateless)
앞에 보낸 데이터를 기억하지 않습니다. 요청에 대한 정보만 보냅니다.

### 요청-응답 구조
- Request와 Response로 나뉘어져 있습니다.
- 하나의 요청에는 반드시 하나의 응답이 돌아옵니다.
- URL을 치면 HTTP GET 요청으로 전송됩니다.

---

## HTTP 요청 구조

### Start Line

```
GET /products HTTP/1.1
```

**구성 요소**
- **GET**: HTTP 메서드 (요청의 목적을 나타냄)
- **/products**: 요청 주소 (resource, RESTful한 설계에서는 행위와 자원으로 표현)
- **HTTP/1.1**: 프로토콜 버전

### Header

Key-Value 쌍으로 추가적인 정보를 담습니다. 보통 여러 개가 들어갑니다.

**주요 헤더**
- **Content-Type**: 바디에 있는 데이터 타입
- **Accept**: 클라이언트가 받고 싶은 데이터 형태

### Body

실제로 전달해야 할 데이터를 담습니다.

- GET 메서드는 보통 바디가 없습니다.
- Key-Value 형태로 필터링, 정렬 정보 등을 담습니다.

---

## HTTP 응답 구조

### Status Line

```
HTTP/1.1 200 OK
```

**구성 요소**
- **HTTP/1.1**: 프로토콜 버전
- **200**: 상태 코드 (Status Code)
- **OK**: 상태 코드 설명 (Reason Phrase)

### Header

Key-Value 쌍으로 추가적인 정보를 담습니다.

### Body

실제 응답 데이터를 Key-Value 형태(DTO)로 담습니다.

**주의**: 204, 304 상태 코드는 바디가 없을 수 있습니다.

---

## 주요 HTTP 메서드

### GET (조회)
- 데이터 조회에 사용
- Body에 내용을 담을 수 없음
- 캐싱 가능
- 멱등성 보장

### POST (생성)
- 데이터 생성에 사용
- 캐싱 불가능
- Body에 데이터를 담기 때문에 상대적으로 보안에 안전 (HTTPS 사용 시 암호화됨)
- 예시: 로그인 (인증 정보 생성)

### PUT (수정/생성)
- 전체 수정 (덮어쓰기)
- 없으면 생성, 있으면 전체를 덮어쓰기
- 멱등성 보장 (여러 번 반복해도 동일한 결과)
- 실무에서 잘 사용하지 않음

### PATCH (일부 수정)
- 일부 필드만 수정
- 모든 요구사항을 바꿀 때도 사용 가능

### DELETE (삭제)
- 데이터 삭제에 사용
- Body에 데이터를 담지 않음
- 멱등성 보장

---

## 멱등성 (Idempotent)

여러 번 반복해도 동일한 결과가 나오는 성질입니다. (해시 함수처럼)

**멱등성을 보장하는 메서드**
- GET
- PUT
- DELETE

**멱등성을 보장하지 않는 메서드**
- POST

---

## 상태 코드 (Status Code)

### 2xx - 성공
- **200 OK**: 요청 성공

### 4xx - 클라이언트 오류
클라이언트의 잘못된 요청

**주요 상태 코드**
- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found

### 5xx - 서버 오류
서버의 오류

**주요 상태 코드**
- 500 Internal Server Error
- 502 Bad Gateway
- 503 Service Unavailable

---

## RESTful 설계

자원(Resource)과 행위(HTTP 메서드)로 API를 표현하는 설계 방식입니다.

**예시**
- `GET /products`: 상품 목록 조회
- `GET /products/1`: 특정 상품 조회
- `POST /products`: 상품 생성
- `PATCH /products/1`: 특정 상품 수정
- `DELETE /products/1`: 특정 상품 삭제

---

## 핵심 정리

- **비연결성**: 한 번 데이터를 주고받으면 연결을 끊음
- **무상태성**: 이전 요청을 기억하지 않음
- **주요 메서드 5가지**: GET(조회), POST(생성), PUT(수정/생성), PATCH(일부 수정), DELETE(삭제)
- **GET의 특징**: Body에 데이터를 담을 수 없음
- **상태 코드**: 2xx(성공), 4xx(클라이언트 오류), 5xx(서버 오류)
- **멱등성**: GET, PUT, DELETE는 멱등성 보장
