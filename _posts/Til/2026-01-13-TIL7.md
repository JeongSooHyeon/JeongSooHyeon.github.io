### 1. 예외 처리 원칙 ⭐

#### 핵심 원칙
> **예외는 받는 쪽에서 처리하는 게 맞음**

- 예외가 발생하는 곳에서 try-catch를 쓰면 어차피 같은 예외를 던지는 것
- 예외를 처리할 수 있는 적절한 위치(받는 쪽)에서 처리해야 함

```java
// ❌ 나쁜 예: 발생하는 곳에서 처리
public void doSomething() {
    try {
        // 예외 발생 가능한 코드
    } catch (Exception e) {
        throw e; // 어차피 다시 던짐
    }
}

// ✅ 좋은 예: 받는 쪽에서 처리
public void caller() {
    try {
        doSomething();
    } catch (Exception e) {
        // 적절한 처리
    }
}
```

### 2. orElse() vs orElseGet() 차이

#### 공통점
- 가져온 값이 null일 때 해당 값을 반환

#### orElse()
- 해당 값이 null이든 아니든 관계없이 **항상 호출됨**
- **값을 인수로 받음** (메서드가 아님)

```java
optional.orElse(defaultValue) // 항상 실행됨
```

#### orElseGet()
- 해당 값이 **null일 때만 호출됨**
- **Supplier(함수형 인터페이스)를 인수로 받음**

```java
optional.orElseGet(() -> defaultValue) // null일 때만 실행됨
```

#### 성능 차이
- `orElseGet()`이 더 효율적 (값이 있을 때는 실행하지 않음)
- 기본값 생성 비용이 클 때는 `orElseGet()` 사용 권장
