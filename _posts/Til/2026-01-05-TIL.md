# 📅 2026년 1월 5일 학습 기록

## 1. JVM (Java Virtual Machine) 구조와 실행 원리 `❗중요`
Java 소스코드(`.java`)가 실행되기까지의 과정과 메모리 구조를 학습했습니다.



### 🏗️ Runtime Data Area (메모리 영역)
* **Method Area:** 메서드 정보, `static` 변수, 상수가 저장되는 공간.
* **Heap Area:** 모든 객체(참조 자료형)가 생성되는 공간. **GC(Garbage Collector)**의 주 관리 대상.
* **Stack:** 메서드 호출 시 생성되는 지역 변수와 매개변수가 저장되는 공간. (LIFO 구조)

### 🧹 GC (Garbage Collector)
* **Stop The World:** GC 실행을 위해 JVM이 애플리케이션 실행을 멈추는 현상.
* **Mark and Sweep:** 참조되지 않는 객체를 식별(Mark)하고 제거(Sweep)하는 기본 과정.
* **Generational GC:** Heap을 Young/Old 영역으로 나누어 관리. 대부분의 객체는 금방 사라지므로 Young 영역을 위주로 효율적인 청소를 진행함.

## 2. 객체지향 프로그래밍 (OOP) 심화

### 🔄 상속(Inheritance) vs 포함(Composition)
* **상속 (Is-A):** 단일 상속만 가능하며 계층 관계가 명확함. 결합도가 높음.
* **포함 (Has-A):** 클래스 안에 다른 클래스 인스턴스를 가지는 것. **상속보다 유연하며 결합도가 낮아 실무에서 권장됨.**

### 🛠️ 오버로딩(Overloading) vs 오버라이딩(Overriding)
| 구분 | 오버로딩 (Overloading) | 오버라이딩 (Overriding) |
| :--- | :--- | :--- |
| **개념** | 한 클래스 내 동일 이름 메서드 중복 정의 | 부모 메서드를 자식 클래스에서 재정의 |
| **조건** | 매개변수 개수나 타입이 달라야 함 | 이름, 매개변수, 반환 타입이 동일해야 함 |
| **특징** | 다형성 (과적) | 다형성 (동적 바인딩) |

* **주의:** `static` 메서드는 오버라이딩이 아닌 **Hiding(숨김)** 처리가 되며, `final` 메서드는 재정의가 불가능함.

## 3. 메모리 누수(Memory Leak) 방지
Java는 GC가 메모리를 관리해주지만, 잘못된 설계로 누수가 발생할 수 있습니다.
1.  **컬렉션 누수:** 사용이 끝난 리스트의 요소는 `remove()` 등으로 정리할 것.
2.  **무분별한 static 변수:** 프로그램 종료 시까지 메모리에 남으므로 주의.
3.  **문자열 관리:** 빈번한 수정 시 불변 객체인 `String` 대신 `StringBuilder` 사용 권장.
