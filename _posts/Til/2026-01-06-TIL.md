# 📅 TIL: Java 객체지향 프로그래밍 핵심 정리

## 1. Object 클래스: 자바의 뿌리
* **개념:** 모든 클래스가 상속받는 최상위 클래스(Root).
* **주요 메서드:** `toString()`, `equals()`, `hashCode()` 등 범용 메서드 제공.
* **⚠️ 주의사항:** * `equals()`를 재정의하면 **`hashCode()`도 반드시 재정의**해야 함.
    * 논리적으로 같은 객체(`equals == true`)라면 해시값도 같아야 하기 때문.

---

## 2. 캡슐화 (Encapsulation) `@@@`
> **정보 은닉과 보호:** 내부 구현을 숨기고 외부의 잘못된 접근을 막는 것.

* **방법 1. 접근 제어자 `@@@`:**
    * `private` (클래스 내부만) → `default` (패키지 내부) → `protected` (상속 관계) → `public` (전체)
    * **원칙:** 일단 `private`으로 막고, 필요할 때만 단계적으로 개방.
* **방법 2. Getter & Setter `@@@`:**
    * 메서드를 통해 필드 값에 접근하고 수정함. (Lombok의 `@Getter`, `@Setter`로 자동화 가능)

---

## 3. 불변 객체 (Immutable Object)
* **특징:** 객체 생성 후 상태를 바꿀 수 없는 객체 (예: `String`, `Integer`, `LocalDate`).
* **장점:** 스레드 안전성(Thread-Safe) 확보, 동시성 문제 예방.
* **final 키워드:** 필드에 `final`을 사용하면 생성자에서 반드시 초기값이 할당되어야 함.

---

## 4. 다형성 (Polymorphism)
* **개념:** 하나의 참조변수로 여러 타입의 객체를 참조하는 것.
* **형변환:**
    * **업캐스팅:** 자식 → 부모 (자동).
    * **다운캐스팅:** 부모 → 자식 (명시적 타입 변환 필요, 위험성 존재).
* **instanceof:** 형변환 가능 여부를 확인하여 런타임 오류 방지.

---

## 5. 추상화 (Abstraction) `@@@@@`
> **핵심:** 객체의 본질(행위)만 남기고 구체적인 사항은 하위 클래스에 위임.

### 5-1. 추상 클래스 (Abstract Class)
* `abstract` 제어자 사용, 미완성 설계도.
* 인스턴스화 불가, 상속을 통한 확장이 목적.
* 데이터(필드) 중심의 추상화나 계층 구조가 명확할 때 사용.

### 5-2. 인터페이스 (Interface) `@@@ 중요`
* **행위 중심 추상화:** "무엇을 할 수 있는가"에 집중.
* 모든 메서드는 `public abstract` (생략 가능).
* **다중 구현** 가능, `default` 메서드를 통해 공통 로직 제공 가능.



---

## 6. 객체지향 5대 원칙 (SOLID) `@@@@`
> **면접 단골 주제!** 유연하고 유지보수가 쉬운 설계를 위한 원칙.

1. **SRP (단일 책임):** 클래스는 하나의 책임만 가질 것.
2. **OCP (개방-폐쇄):** 확장엔 열려 있고 수정엔 닫혀 있을 것.
3. **LSP (리스코프 치환):** 자식 클래스는 부모 클래스의 기능을 완전히 대체 가능해야 함.
4. **ISP (인터페이스 분리):** 사용하지 않는 메서드에 의존하지 않도록 인터페이스를 작게 쪼개기.
5. **DIP (의존성 역전) `@@`:** 구체적인 클래스보다 **인터페이스(추상화)**에 의존할 것. (생성자 주입 활용)

---

## 7. 내부 클래스 (Inner Class)
* **응집도 향상:** 특정 클래스 안에서만 사용되는 클래스를 내부에 선언.
* **정적 내부 클래스 (static):** 외부 객체 없이 생성 가능, 외부 클래스의 static 필드만 접근 가능.
* **지역 내부 클래스:** 메서드 내부 선언, `effectively final` 변수만 접근 가능.
* **익명 내부 클래스:** 이름 없이 즉석에서 생성하는 일회용 클래스. (함수형 인터페이스 구현 시 활용)
